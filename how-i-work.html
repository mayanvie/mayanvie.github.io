Approach

How I Work

Day one is emotional logistics as much as information. I aim for clarity people will actually open, and practice they'll try before stakes go up — whether that's a guide's first week or a program that has to run again next month. For artifacts and case studies, see Work.

step 01

Start with context

Who's in the room, what they already carry, and the questions they're too embarrassed to ask in week one.

step 02

Sequence the week

Orientation before detail — a spine for the first days so nobody meets a wall of new information at hour one.

step 03

Write for doing

Plain-language guides, rehearsals, and a lightweight check for “can you do it yet” — not slide decks alone.

step 04

Keep the rails visible

Trackers, reminders, and handoffs — the unglamorous layer that stops things from quietly falling apart.

step 05

Edit where friction shows

If the same question comes up twice, that's usually the next paragraph — not a new tool.

Systems

LMS & corporate systems

Honestly — I haven't run an enterprise LMS in production yet. But I already think like an ops teammate: clear pathways, enrollments, completion signals, and an experience that answers “what do I do today?” without a scavenger hunt.

I'm ready to learn tools like iLearn (or similar) quickly. The logic isn't new — sequenced content, practice tasks, pass rules. The interface is.

On my Substack I write about input vs. output — consuming content alone rarely builds skill. I build the same way: rehearsal early, even when it's messy. See essays.
